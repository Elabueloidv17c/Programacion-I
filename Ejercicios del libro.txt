1.-¿Qué es software?
	Una colección de programas corriendo en una computadora


2.- ¿Por qué es importante?
	Porque está en millones de computadoras en todo el mundo, todos estamos rodeados de ellas y las usamos en nuestro día a día


3.- ¿Dónde es importante el software?
	En muchos lugares, desde un reloj o una calculadora, hasta barcos de transporte, las turbinas de un avion o incluso las
	transmisiones de la NASA


4.- ¿Qué puede pasar si un software falla?
	La gente puede enojarse o incluso salir lastimada, para ejemplificar esto último:
	a) Si el software de un avión falla en pleno vuelo, esto podría provocar que el avión pierda el control.
	b) Si el software de un CAT falla, esto podría resultar en la muerte del paciente.


5.- ¿Dónde juega un papel importante el software?
	En cualquier computadora, tales como nuestro celular, televisor, tablet, laptop, PC, Automovil o calculadora.
	Además de esto, las computadoras están presentes en el transporte local, nacional e internacional, en autobuses,
	trenes, aviones o barcos.

7.-¿Cuál es la diferencia entre ciencia computacional y programación?
	La programación es solo una rama de la ciencia computacional, está dedicada exclusivamente a la escritura de código.
	La ciencia computacional es una ciencia que estudia sistemas computacionales, usa teorías de entendimiento
	de sistemas computacionales y metodos de diseño, algoritmos, herramientas, además de metodos de testeo, 
	análisis o verificación, todo esto tanto en el lado teorico como en la implementacion.


8.- ¿Para qué se usa el software en el diseño, construcción y uso de un barco?
	Diseño: El diseño general de la máquina (incluyendo el motor) son diseñados usando un software, los planos, los calculos y las
	simulaciones de rendimiento de cada parte.
	Construcción: El ensamble del barco se calcula minusiosamente por un software, una vez listo, la implementacion de esto
	la controla un software.
	Uso: Un barco está lleno de computadoras que requieren un software para funcionar, empezando por el motor, el manejo de las 
	rutas de entrega, la capacidad de almacenamiento, etc.

3.-¿Funcion que debe aparecer en cualquier prograa c++?
	La funcion main

4.- ¿Cuál es el proposito de return 0;?
	Es el valor de retorno, la funcion main es de tipo entero, por tanto es necesario regresar un numero entero como resultado de la misma

5.- ¿Cuál es el proposito de un compilador?
	Es el encargado de convertir el código fuente (lo que el programador escribe) en un lenguaje que la computadora pueda entender y ejecutar

6.-¿Cuál es el proposito de #include?
	Sirve para incluir lineas de código externas a código local, con el proposito de añadir funciones extra al mismo.

7.-¿Qué es el sufijo .h en un archivo de c++?
	Es el tipo de archivo del que provienen las librerías que se pueden importar con #include, estos archivos contienen funciones extra para nuestro código

8.-¿Qué hace un linker?
	Conecta el código escrito por nosotros con las librerías externas para que funcionen en el programa final

9.-¿Cuál es la diferencia entre código fuente y archivo fuente?
	El código fuente es lo que el programador escribe, el archivo fuente es el resultado de la compilación de este código

10.- ¿Qué es un IDE y para qué sirve?
	Es un editor de tecto que tiene herramientas que ayudan a facilitar la escritura de código, así como su compilacion

11.-Si entiendo todo lo del libro ¿necesito practicar?
	Claro que sí, de nada sirve entender la teoría si no se practica, es así cuando realmente aprendes, ya que te encuentras con problemas y buscas soluciones.
	ese es el verdadero trabajo de un programador

5.-Terminos:

	//: Es el termino que se usa para que el compilador ignore el texto que hay después de él, esto sirve para escribir comentarios para los programadores
	<<: Es el termino que se utiliza para indicarle al compilador que el texto siguiente es un parametro de salida, se escribe después de un "cout"
	C++: Es un lenguaje de programación en el que una persona escribe instrucciones para una computadora
	Comentario:	Es el texto que el compilador ignora, está hecho para que los programadores se guien
	Compile-time error: Es el error que un IDE te devuelve cuando hay un error al intentar compilar el código
	Cout:	Es la palabra reservada de la librería iostream que sirve para expresar parametros de salida
	Executable: Es el archivo que resulta de la compilación, sirve para correr el programa
	Funcion: Es un conjunto de instrucciones diseñado para hacer una tarea en específico, puede ser "llamado" por el programador cuantas veces quiera
	Header: Es una directriz que sirve para incluir archivos externos al codigo que hacen accesibles funciones extra al programa
	IDE: Es un editor de tecto que tiene herramientas que ayudan a facilitar la escritura de código, así como su compilacion
	#include: Es la sintaxis para incluir librerías al código
	Librería: Es una serie de funciones externs que pueden ser añadidas al código para añadir funciones
	Linker:	Es la parte que se encarga de conectar el código local con las librerías externas y así compilar todas las funciones del mismo en el ejecutable
	main (): Es la funcion que se ejecuta primero en un programa en c++, de hecho es obligatorio incluirla en nuestros códigos
	Source Code: Es el código que el programador escribe de manera local
	Output:	Es Cualquier tipo de dato de salida que se muestea por pantalla.
	Source Code: Es el código que un humano puede entender, es decir, es lo previo a la traducción a lenguaje de máquina
	Programa: Es el resultado del proceso de compilación, dicho de otra forma, es el programa terminado listo para ser usado por la gente
	Declaracion: Es el nombre que se le da al momento en el que se crea cualquier variable, función o argumento, sirve para asignarle un espacio en la memoria.
